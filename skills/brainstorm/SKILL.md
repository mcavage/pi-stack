# brainstorm

Turn a fuzzy idea into something you could start coding tomorrow. Collaborative
and generative, with **no VC interrogation** ("TAM? wedge? moat? desperate
customer?"). This is for builders, not fundraising.

## Flow
1. **Understand.** Reflect the idea back in a sentence or two. Ask only what you
   genuinely need to proceed: 1-3 questions, max.
2. **Explore.** Surface 2-3 distinct approaches with honest trade-offs, and note
   prior art / similar tools so we're not reinventing.
3. **Shape.** Pick a direction (or let the user pick), then define: the core user
   experience, the smallest version worth building, the main technical pieces,
   and the real risks / unknowns.
4. **Write.** Save a `DESIGN.md`: problem, chosen approach, MVP scope,
   architecture sketch, open questions. Tight and buildable, not a treatise.

Bias toward building. The output is a doc, not a deck.
